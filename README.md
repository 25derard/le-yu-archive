# le-yu-archive

## 项目简介

`le-yu-archive` 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 页面的集合存放地，便于长期保存与访问。

## 目录说明

```
le-yu-archive/
├── pages/           # 存放所有归档的 HTML 页面
├── assets/          # 页面依赖的静态资源（CSS、JS、图片等）
├── index.html       # 仓库首页（可浏览页面列表）
└── README.md        # 本文件
```

- `pages/` 目录下每个子文件夹或文件对应一个归档页面，命名简洁明了。
- `assets/` 目录用于存放页面所需的公共资源，避免重复引用。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，不依赖外部网络资源（除非在页面内明确声明）。每个页面均保持原始结构与样式，仅做必要的本地化调整以确保在浏览器中正常显示。

归档页面采用以下原则：

- 不修改页面核心内容与功能。
- 仅调整资源引用路径，使其指向本仓库的 `assets/` 目录。
- 保留页面原有的元数据（如标题、描述、编码声明等）。

## 维护说明

本仓库由维护者不定期更新，新增页面或修复现有页面问题。如有以下需求，欢迎通过 Issues 或 Pull Requests 参与：

- 提交新的归档页面（请附上原始来源说明）。
- 报告页面显示异常或链接失效。
- 建议优化目录结构或页面组织方式。

请注意，本仓库仅作为归档用途，不提供任何形式的托管、推广或担保。页面内容版权归原始作者所有，本仓库仅做技术性存档。

## 使用方式

你可以直接通过 GitHub Pages 或本地浏览器打开 `index.html` 浏览页面列表。也可以单独访问 `pages/` 目录下的任意 HTML 文件。

如需将整个仓库克隆到本地：

```bash
git clone https://github.com/your-username/le-yu-archive.git
```

然后直接在浏览器中打开 `index.html` 即可。
